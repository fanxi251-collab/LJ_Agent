# LingJing AI 项目部署运行配置说明书

当前时间：2026-07-19 21:08:04 +08:00（Asia/Shanghai）

适用项目版本：当前工作区 `LingJing_AI 0.1.0`

## 1. 文档范围与结论

本文基于当前工作区的后端源码、前端源码、测试、脚本、配置读取逻辑和已有架构文档编写，适用于开发环境、演示环境和单机生产部署。

项目当前没有 Dockerfile、Compose、Nginx、systemd 或 Windows 服务配置。推荐部署形态是：

```text
浏览器
  -> HTTPS 反向代理（生产环境建议）
  -> 单个 Uvicorn worker / FastAPI :8000
       -> Vue 构建产物、管理端静态页面、Live2D 资源
       -> 本地 Qdrant：qdrant_db/
       -> SQLite：data/conversations.db、data/attractions.db
       -> 阿里云百炼：问答、Embedding、Qwen-Audio Realtime
       -> 高德：天气、地点、路线、前端地图
       -> Redis：可选缓存
       -> Neo4j：可选知识图谱
```

部署时必须注意以下约束：

- 必须从项目根目录启动。`AppSettings.for_workspace(Path.cwd())` 会把当前目录作为数据、日志、Prompt 和 Qdrant 的根目录。
- 应从完整源码目录运行并执行 editable install。当前 Python 包不会把 `frontend/` 一并打入 wheel，不能只复制安装后的 `site-packages/lingjing_ai`。
- 本项目使用嵌入式本地 Qdrant，同一个 `qdrant_db/` 只能由一个应用进程可靠持有；生产环境使用 `--workers 1`，不要多 worker 共享该目录。
- `pyproject.toml` 只声明了部分 Python 依赖，仅执行 `pip install -e .` 不足以启动 FastAPI。
- 游客端数字人需要浏览器麦克风。远程部署必须使用 HTTPS；`localhost` 本地开发可直接使用 HTTP。
- 当前管理端接口没有登录、鉴权和权限控制。生产环境不可直接暴露到公网，必须由反向代理、VPN、内网或额外鉴权层保护。

## 2. 组件、端口与外部依赖

| 组件 | 默认地址/端口 | 是否必需 | 作用 |
| --- | --- | --- | --- |
| FastAPI/Uvicorn | `127.0.0.1:8000` | 是 | API、游客端构建产物、管理端页面、WebSocket |
| Vite 开发服务器 | `127.0.0.1:5173` | 仅前端开发 | 热更新；`/api` 和 `/static` 代理到 8000 |
| Vite Preview | `127.0.0.1:4173` | 否 | 预览构建产物，不是正式生产服务器 |
| Qdrant 本地库 | `qdrant_db/` | 是 | RAG 向量检索，无独立端口 |
| SQLite | `data/*.db` | 是 | 会话、景点及图片元数据 |
| Redis | 配置项决定，常见为 6379 | 否 | 问答和高德结果缓存；失败时自动降级 |
| Neo4j | Bolt 常见为 7687 | 否 | 知识图谱增强；失败时自动禁用 |
| 阿里云百炼 | HTTPS/WSS 443 | 完整能力必需 | LLM、Embedding、实时语音 |
| 高德开放平台 | HTTPS 443 | 地图能力必需 | Web 服务和浏览器 JS 地图 |

生产环境对外通常只开放 80/443，Uvicorn、Redis 和 Neo4j 应仅监听本机或受控内网。

## 3. 环境要求

### 3.1 已验证版本

当前工作区验证环境为：

| 软件 | 已验证版本 |

### 3.2 磁盘与权限

运行账号至少需要对以下目录具有读写权限：

- `data/`：文档、SQLite、景点图片、分析快照。
- `qdrant_db/`：本地 Qdrant 数据和锁文件。
- `logs/`：问答和 Agent 日志。
- `reports/qa_eval/`：仅运行评测时需要。

前端源码、`frontend/dist/`、`frontend/static/`、`frontend/public/digital-human/`、`prompt/` 和 `config/` 至少需要读取权限。

## 6. 配置加载规则

配置读取顺序由低到高为：

```text
代码默认值 < 项目根目录 .env < 项目根目录 config.yml < 操作系统环境变量
```

同名项最终以操作系统环境变量为准。建议遵循：

- `config.yml` 只放模型名称、开关、TTL 等非敏感配置。
- API Key、密码、Redis URL 优先放操作系统环境变量或密钥管理服务。
- 开发机可把密钥放 `.env`，但不要在 `config.yml` 中重复同名项，否则 `config.yml` 会覆盖 `.env`。
- `.env` 和 `config.yml` 已被 `.gitignore` 忽略，不应提交真实密钥。

布尔值支持 `1`、`true`、`yes`、`on` 或 `启用`；其他值会被识别为关闭。

### 7.3 配置项说明

#### 阿里云与 Embedding

| 配置项 | 默认值 | 说明 |
| --- | --- | --- |
| `LJAPI_KEY` | 空 | LLM、默认 Embedding、实时语音共用的百炼 Key |
| `LJ_LLM_MODEL` | `qwen3.7-max` | 普通回答模型 |
| `LJ_LLM_BASE_URL` | DashScope compatible-mode | OpenAI 兼容 HTTP 根地址 |
| `LJ_EMBEDDING_PROVIDER` | `aliyun` | 有 Key 时调用阿里云；无 Key 时自动使用本地 Hashing Embedding |
| `LJ_EMBEDDING_MODEL` | `text-embedding-v4` | 向量模型 |
| `LJ_EMBEDDING_DIMENSIONS` | `1024` | 向量维度，必须与当前 collection 一致 |
| `LJ_EMBEDDING_API_KEY_ENV` | `LJAPI_KEY` | 指定 Embedding Key 所在的配置名 |
| `LJ_REALTIME_MODEL` | `qwen-audio-3.0-realtime-flash` | 实时文本/语音模型 |
| `LJ_REALTIME_WORKSPACE_ID` | 空 | 配置后使用北京地域专属域名 |
| `LJ_REALTIME_URL` | 空 | 完整 WSS 地址，优先级高于自动生成地址 |
| `LJ_REALTIME_HISTORY_TURNS` | `6` | 向上游重注入的最近轮数，范围被限制为 1—50 |
| `LJ_REALTIME_CONNECT_TIMEOUT_SECONDS` | `15` | 上游 WSS 连接超时 |

没有 `LJAPI_KEY` 时，服务仍可使用本地抽取式回答和 Hashing Embedding 启动，但无法获得完整 LLM、问题扩写和数字人实时语音能力。

切换 Embedding provider、模型或维度后必须停止后端并重建向量库。同维度下更换模型不会自动识别语义空间变化；不要继续复用旧向量。

#### 高德

| 配置项 | 作用 |
| --- | --- |
| `MAP_API` | 后端天气、地点和路线 Web 服务 Key |
| `MAP_JS_API` | 浏览器端地图 JS API Key |
| `MAP_JS_SECURITY_CODE` | 与 JS API Key 配套的安全密钥 |
| `AMAP_ROUTE_DEFAULT_MODE` | 默认路线方式，当前推荐 `driving` |
| `AMAP_SCENIC_NAVIGATION_RADIUS_KM` | 景区导航安全边界，默认 10 km |

`MAP_JS_API` 和安全密钥会通过 `/api/tools/map/config` 返回给浏览器，这是前端地图的工作方式。必须在高德控制台设置允许域名等调用限制，不能把该 Key 当作服务器端秘密。

#### Redis

Redis 是性能增强项。`REDIS_ENABLED=true` 且 `REDIS_URL` 可连接时启用；客户端未安装、连接失败或运行中断开时会自动退回内存缓存，不阻止应用启动。

推荐检查：

```powershell
redis-cli -u "redis://:替换为密码@127.0.0.1:6379/0" ping
```

预期返回 `PONG`。生产环境不要把真实 Redis URL 写入命令历史。

#### Neo4j

`KG_ENABLED=true` 时需要 `NEO4J_URI`、`NEO4J_USER` 和 `NEO4J_PASSWORD`。驱动缺失、配置不全或连接失败时知识图谱会降级为关闭状态，不影响基础 RAG 启动。

检查接口：

```text
GET /api/admin/knowledge-graph/status
```

#### Web 搜索预留项

代码可以读取 `WEB_SEARCH_API_URL`、`WEB_SEARCH_API_KEY_ENV` 和后者指定的密钥（默认名为 `SEARCH_API_KEY`），但当前 `agent_use_web_search` 是代码内固定关闭的能力。仅填写这些配置不会启用外部 Web 搜索；在没有同步修改和测试业务代码前，应保持为空。

## 8. 数据初始化

### 8.1 全新知识库

全新部署没有现成 `qdrant_db/` 时，应用会创建本地 collection。将 UTF-8 编码的 `.md` 或 `.txt` 文档放入知识库的推荐方式是：

```powershell
python scripts/import_document.py "待导入资料.md" --workspace .
```

也可启动后通过管理端上传：

```text
http://127.0.0.1:8000/admin/documents
```

单个资料仅支持 `.md`、`.txt`、UTF-8 编码，最大 5 MiB。

### 8.2 重建向量库

执行前必须停止 Uvicorn、在线评测和其他打开同一 `qdrant_db/` 的进程：

```powershell
python scripts/rebuild_vector_store.py --workspace .
```

向量维度变化时，代码会保留旧 collection，并使用名称带 `_dim_维度` 的新 collection；它不会删除旧 collection。更换同维度的 Embedding 模型时也应主动运行重建脚本。

### 8.3 重建知识图谱

确认 Neo4j 已启用并可连接后执行：

```powershell
python scripts/rebuild_knowledge_graph.py --workspace .
```

### 8.4 初始化游客分析快照

管理端分析页依赖固定 JSON 快照。缺少或损坏时，该接口返回 503，但不会阻止后端其他功能启动：

```powershell
python scripts/build_tourism_analytics_snapshot.py `
  --input "景点景区旅游数据行为分析数据.xlsx" `
  --output "data/tourism_analytics_snapshot.json"
```

如果部署包已包含有效的 `data/tourism_analytics_snapshot.json`，运行环境不需要保留原始 Excel。

### 8.5 景点数据库

`data/attractions.db` 为空或不存在时，首次启动会自动创建表并播种 8 个默认景点，同时把 `src/lingjing_ai/assets/attractions/seed-*.webp` 复制到 `data/attraction_images/`。管理员上传的自定义封面不会被自动覆盖。

### 8.6 美食与反馈数据库

`data/foods.db` 为空或不存在时，首次启动会自动创建表并播种 6 条美食推荐，同时把 `src/lingjing_ai/assets/foods/seed-*.webp` 复制到 `data/food_images/`。`data/feedback.db` 独立保存匿名游客反馈和管理回复；两者都不修改景点、会话、RAG 或语音数据结构。

### 9.3 生产运行

生产环境不要使用 `--reload`，也不要增加 worker：

```powershell
python -m uvicorn lingjing_ai.api.main:app `
  --host 127.0.0.1 `
  --port 8000 `
  --workers 1 `
  --proxy-headers `
  --forwarded-allow-ips 127.0.0.1
```

若不使用反向代理且必须在受控局域网访问，可改为 `--host 0.0.0.0`。这样会同时暴露未鉴权的管理 API，且非 localhost 的 HTTP 页面通常无法使用麦克风，因此不推荐作为公网方案。

## 10. HTTPS 反向代理要求

生产环境反向代理必须同时支持：

- 普通 HTTP API。
- SSE 流式接口，关闭代理缓冲。
- `WS /api/visitor/realtime` WebSocket 升级。
- 长回答和实时语音的较长读取超时。
- 至少 5 MiB 上传；建议代理限制设为 6 MiB。
- `/visitor/guide`、`/visitor/explore`、`/visitor/map` 原样转发到 FastAPI，由后端返回 Vue 入口。

Nginx 参考片段：

```nginx
map $http_upgrade $connection_upgrade {
    default upgrade;
    ''      close;
}

server {
    listen 443 ssl http2;
    server_name example.com;

    client_max_body_size 6m;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $connection_upgrade;
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;
        proxy_buffering off;
    }
}
```

证书和私钥路径应按实际证书管理方式配置。浏览器端会根据页面协议自动使用 `ws://` 或 `wss://`，不需要单独写前端环境变量。

## 11. Linux systemd 参考

下面示例仍保持单 worker；路径、用户和虚拟环境必须替换为实际值：

## 13. 日常维护与备份

### 13.1 备份

为获得一致快照，先停止 Uvicorn，再备份第 4.2 节列出的持久化数据。另行备份：

- 生产环境变量或密钥管理记录。
- `config.yml` 的非敏感配置。
- `config/asr_glossary.yml` 的人工词典。
- Neo4j 官方备份文件（若启用）。

Redis 只承担缓存，通常不纳入业务恢复目标。

### 13.2 恢复

恢复时应使用同一组 `data/uploaded/`、manifest 和 Qdrant 数据，并确保 `LJ_EMBEDDING_MODEL`、`LJ_EMBEDDING_DIMENSIONS` 与备份时一致。恢复后先以单进程启动并执行第 12 节检查。

### 13.3 资料和模型变更

- 新增单个资料：使用管理端或 `import_document.py`。
- 批量资料状态重建：停止后端后运行 `rebuild_vector_store.py`。
- 修改知识图谱抽取规则：重建 Neo4j 图谱。
- 修改 Embedding 模型或维度：重建全部向量。
- 修改 Vue：重新执行 `npm run test` 和 `npm run build`。
- 修改 `config/asr_glossary.yml`：运行中会按 TTL 自动重新加载，通常无需重启。

## 14. 常见故障

### 14.1 `ModuleNotFoundError: lingjing_ai`

原因通常是没有在虚拟环境执行 editable install，或不在项目根目录运行。

```powershell
python -m pip install -e .
python -c "import lingjing_ai; print(lingjing_ai.__file__)"
```

### 14.2 FastAPI 启动时报 multipart 依赖缺失

```powershell
python -m pip install python-multipart
```

### 14.3 `/visitor` 显示“Vue 游客端尚未构建”

```powershell
Set-Location frontend
npm ci
npm run build
Set-Location ..
```

确认 `frontend/dist/index.html` 存在后刷新页面。

### 14.4 Qdrant 锁、数据库占用或启动失败

确认没有另一个 Uvicorn、测试、评测或重建脚本正在打开同一个 `qdrant_db/`。不要用多个 worker，也不要在后端运行期间执行 benchmark 或重建向量库。

### 14.5 问答能启动但检索质量异常

检查是否在已有阿里云向量上改用了本地 Hashing Embedding，或更换了同维度的 Embedding 模型。统一配置后停止后端并重建向量库。

### 14.6 Redis 不可用

Redis 故障不会阻止启动。检查 `REDIS_URL`、网络和认证；若暂时不使用，设为：

```yaml
REDIS_ENABLED: false
```

### 14.7 Neo4j 状态为关闭

检查 `KG_ENABLED`、连接参数、Neo4j Python 驱动和服务状态。基础 RAG 可在图谱关闭时继续运行。

### 14.8 数字人无法连接或没有音频

按顺序检查：

1. `LJAPI_KEY`、Realtime 模型、Workspace ID 或自定义 URL。
2. 服务器能否访问外部 WSS 443。
3. 反向代理是否传递 WebSocket Upgrade 头。
4. 页面是否为 HTTPS 或 localhost。
5. 浏览器是否授予麦克风权限。
6. `/digital-human/pcm-capture-worklet.js` 和 Live2D 模型资源是否返回 200。

### 14.9 地图显示空白

检查 `MAP_JS_API` 和 `MAP_JS_SECURITY_CODE` 是否同时配置、高德控制台允许域名是否匹配，以及 `/api/tools/map/config` 是否返回 `enabled=true`。路线和天气还需要独立的 `MAP_API`。

### 14.10 游客分析接口返回 503

重新生成 `data/tourism_analytics_snapshot.json`。该错误只影响管理端分析页，不影响游客问答。

## 15. 安全上线清单

- 不在 Git、日志、截图或说明书中保存真实 API Key、Redis URL、Neo4j 密码。
- 若密钥曾以明文出现在共享文件中，先在对应平台轮换，再部署新密钥。
- 对高德 JS Key 配置域名限制，对服务端 Key 配置适用的调用限制。
- 仅通过 HTTPS 提供远程游客端和数字人功能。
- Uvicorn 只监听回环地址，由反向代理对外服务。
- 管理页面及 `/api/admin/*`、上传、重建、景点编辑接口增加访问控制。
- 对 `data/`、`qdrant_db/`、日志和密钥文件设置最小文件权限。
- 反向代理限制上传大小、请求频率和来源；应用自身的文档和图片上传上限均为 5 MiB。
- 备份前停止后端，定期执行恢复演练。
- 保留 Live2D 和 Cubism 资源附带的 LICENSE、NOTICE 与 README。
