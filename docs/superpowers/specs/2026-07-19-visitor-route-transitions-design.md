# 游客端“湖光溶解”页面切换设计

当前时间：2026-07-19 Asia/Shanghai

## 目标

为 `/visitor/guide`、`/visitor/explore` 与 `/visitor/map` 增加统一、克制且具有景区气质的页面切换效果。桌面侧栏和移动底栏在切换过程中保持固定，仅业务内容区使用“湖光溶解”动效；三个页面在当前浏览器会话中保留各自的交互状态和滚动位置。

## 范围

- 只调整游客端共享应用外壳、路由视图承载方式和支持缓存所必需的页面生命周期。
- 不修改三个页面的业务布局、视觉内容、后端 API、WebSocket 协议或数据结构。
- 不为管理端增加路由过渡。
- 不引入新的动画依赖；路由切换使用 Vue `Transition`、`KeepAlive` 与 CSS 实现，现有 GSAP 继续只服务电影式开场。
- 视觉验收不调用真实 AI、实时语音或付费地图路线服务。

## 已确认的体验方向

- 全局导航固定，只有右侧内容区切换。
- 导游、探索、地图三个页面均保留状态。
- 动效采用“湖光溶解”：淡入、微升、轻微失焦和极轻缩放共同形成平静的呼吸感。
- 不采用方向性横向推镜，也不采用每次切换都出现的水波揭幕。

## 组件架构

### 共享应用外壳

`App.vue` 继续拥有游客端侧栏、移动底栏和导游开场。`RouterView` 改为插槽形式，将当前路由组件与 `route.path` 交给独立的 `VisitorRouteTransition`。导航仍位于过渡组件之外，因此不会随页面位移或重新挂载。

首次加载不播放路由进入动画，避免与电影式开场叠加；只有三个游客页面之间发生真实路径切换时才播放“湖光溶解”。仅查询参数变化不会重新创建页面，也不会播放整页过渡。

### `VisitorRouteTransition`

新增职责单一的共享组件，接口如下：

- `component: Object | Function`：`RouterView` 提供的当前页面组件。
- `routeKey: string`：使用 `route.path`，不使用 `route.fullPath`。
- `routeTheme: "guide" | "default"`：控制内容区背景交叉淡化，避免摄影背景与普通页面底色突然闪切。

组件内部使用一个最大容量为 `3` 的 `KeepAlive` 缓存和一个 Vue `Transition`。进入页与离开页短暂重叠，不使用 `out-in` 模式，因此中间不会出现空白帧。过渡中的页面由同一个内容舞台约束，不改变导航和应用外壳尺寸。

### 路由内容舞台

共享内容区改为固定高度的相对定位舞台。每个页面根节点承担自己的纵向滚动，从而在缓存节点被停用和重新激活后保留原有 `scrollTop`。导游页继续使用视口高度布局；探索页和地图页保持现有内容宽度、间距与响应式规则。

舞台包含随路由交叉淡化的背景表面：导游主题复用当前摄影背景、焦点和渐变，默认主题复用当前米白底色。该背景表面只解决切换期间的闪白，不改变页面既有视觉值。

## 动效规范

### 进入

- 透明度：`0` 到 `1`。
- 垂直位移：`14px` 到 `0`。
- 模糊：`7px` 到 `0`。
- 缩放：`0.994` 到 `1`。
- 时长：`340ms`。
- 缓动：平缓的自定义 cubic-bezier，前段快速脱离静止、后段柔和收束。

### 离开

- 透明度：`1` 到 `0`。
- 垂直位移：`0` 到 `-6px`。
- 模糊：`0` 到 `4px`。
- 缩放只允许极轻变化，不产生可感知的页面尺寸跳动。
- 时长：`220ms`。

进入与离开共享合成层，只动画 `opacity`、`transform` 和短时 `filter`。过渡结束后移除 `will-change`，避免三个缓存页面长期占用 GPU 图层。连续快速点击导航时，Vue 过渡收束当前节点并以最后一次路由为准，不建立动画队列，也不阻止用户继续导航。

## 状态与数据流

### 导游页

- 缓存当前会话、消息、输入内容、数字人模式与所选角色。
- 离开页面时停止隐藏页面的麦克风采集和语音播放，防止用户看不到页面时仍录音或发声。
- WebSocket 保持连接，使已经开始的文字回答可以完成；返回后展示最新文本状态。
- 历史抽屉的 `Escape` 监听改为激活时注册、停用时移除。
- 每次激活时检查非空 `q` 参数；先从 URL 清除该参数，再调用现有 `ask()`，从而保证同一问题只发送一次。

### 探索页

- 景点数据只在首次挂载时加载。
- 缓存关键词、分类、已选择景点、详情抽屉和页面滚动位置。
- 返回页面时不重复请求景点列表，也不自动回到顶部。

### 地图页

- 缓存高德地图实例、景点列表、当前选中项、路线模式、路线摘要和详情抽屉。
- 组件重新激活后在 `nextTick` 调用地图 `resize()`，修复隐藏容器恢复时的尺寸错位。
- 每次激活时检查 `attraction` 参数；若目标景点已加载，则仅更新选中项和地图中心，不重新初始化地图。
- 页面最终卸载时仍调用现有 `destroy()` 释放地图实例。

## 焦点与可访问性

- 过渡期间离开页面设置为不可交互，防止键盘焦点进入正在淡出的控件。
- 过渡完成后焦点移动到新页面的 `<main>`，页面根节点使用临时 `tabindex="-1"` 接收程序化焦点，不进入正常 Tab 顺序。
- 焦点移动使用 `preventScroll`，不破坏缓存恢复后的滚动位置。
- 用户启用 `prefers-reduced-motion: reduce` 时，取消位移、模糊和缩放，仅保留约 `100ms` 的透明度变化。
- 不支持 CSS `filter` 的浏览器自动退化为透明度与轻微位移过渡。

## 异常与边界处理

- 快速切换：最终路由获胜，过渡钩子必须清理旧节点的临时交互状态和合成层提示。
- 查询参数变化：路径未变化时不触发整页动画，只由对应缓存页面消费参数。
- 地图尚未完成加载：激活时的 `resize()` 安全返回；地图初始化完成后仍执行原有视野设置。
- 语音正在录制：离开导游页时终止采集并清理播放，不让麦克风在后台保持活动。
- AI 正在生成：文字流可以在后台完成，返回导游页时继续展示结果；不额外发送问题。
- 电影式开场可见时：底层共享外壳继续保持 `inert`，路由过渡不播放，避免两套动画叠加。

## 测试策略

### 自动化契约

- `App.vue` 使用 `RouterView` 插槽，并把导航保留在 `VisitorRouteTransition` 外部。
- 过渡组件使用 `KeepAlive`，容量为 `3`，缓存键为 `route.path`。
- “湖光溶解”包含确认的进入、离开参数和减少动态效果规则。
- 三个页面分别拥有独立滚动容器，查询参数变化不会改变缓存键。
- 导游页在激活与停用时正确管理键盘、录音和播放；地图页激活时重新测量地图。

### 单元与组件行为

- 首次渲染不播放整页进入动画。
- 三个路径之间切换时只执行一次进入和一次离开。
- 快速连续切换后只保留最终页面，离开页不可交互状态得到清理。
- `prefers-reduced-motion` 下不使用位移、模糊或缩放。
- 返回探索页保留筛选和滚动位置，且不重复获取数据。
- 返回地图页保留路线并调用一次 `resize()`。
- 缓存导游页消费新 `q` 时只发送一次，隐藏期间不继续录音或播放。

### 浏览器验收

- 视口：`1520×856`、`768×1024`、`390×844`。
- 路径：导游 → 探索 → 地图 → 导游，并进行快速连续点击。
- 检查导航固定、背景无闪白、内容无重叠残影、无横向溢出。
- 检查探索筛选与滚动、地图选择与路线、导游消息与数字人状态均能恢复。
- 检查移动底栏始终可点击，页面内容不被底栏遮挡。
- 检查控制台无 Vue 过渡、缓存、地图尺寸或焦点错误。

## 验收标准

- 三个游客页面切换具有一致的“湖光溶解”效果，导航位置始终稳定。
- 切换期间没有空白帧、摄影背景闪切、内容横向跳动或残留页面。
- 三个页面返回后恢复各自状态和滚动位置，不因查询参数变化重建。
- 隐藏的导游页不会继续录音或播放语音，文字回答可以安全完成。
- 缓存地图恢复后尺寸正确，已有路线和选中景点仍然可见。
- 减少动态效果模式下页面仍能清晰、快速、无位移地切换。
- 不修改管理端、不改变后端接口、不覆盖当前 Live2D、实时语音和电影式开场实现。
