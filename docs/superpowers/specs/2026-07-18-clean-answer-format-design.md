# 实时回答格式符清理设计

## 背景

常规 RAG 回答会通过 `answer_formatter.py` 清理 Markdown 行内标记，但实时问答直接在 `answer_contract.finalize_answer()` 中收口。模型返回 `**游览建议**` 等内容时，最终事件和历史记录会保留星号，前端按纯文本渲染后便会显示原始格式符。

## 目标

- 在实时回答最终收口时移除 Markdown 行内格式符。
- 将 `**游览建议**：` 转换为 `游览建议：`。
- 同时清理粗体、斜体、删除线、行内代码和 Markdown 链接的包装符号。
- 保留 `###` 标题和 `-` 列表结构，继续兼容现有游客端分块渲染。
- 清理后的文本用于最终事件与历史持久化，避免刷新历史后再次出现格式符。

## 方案

在 `rag/answer_formatter.py` 增加无状态的 `clean_inline_markdown(text: str) -> str`，只清理行内 Markdown，不改变换行、标题和列表前缀。`realtime/answer_contract.py` 在所有模型回答进入路线补全、长度补全和持久化之前调用该函数。

提示词仍负责约束输出风格，但确定性清理作为最终保障，避免依赖模型完全遵守格式要求。

## 边界

- 不引入 Markdown 渲染依赖，也不使用 `v-html`。
- 不改变现有 `###`、`-` 和 `依据：` 的前端解析协议。
- 实时流中的中间增量可能短暂包含模型原始字符；`assistant.text.done` 会以清理后的完整答案替换页面内容，历史记录只保存清理结果。

## 验证

- 单元测试覆盖用户给出的 `**游览建议**` 示例。
- 覆盖粗体、斜体、删除线、行内代码、链接和结构化标题/列表。
- 运行回答格式器、实时回答契约及实时会话测试。
